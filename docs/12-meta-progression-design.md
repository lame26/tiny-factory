# 메타 진행 설계

## 문서 목적

이 문서는 스테이지를 넘어 유지되는 장기 성장 시스템을 정의한다. 현재 범위는 플레이어 고유 작업자, 보상 상자, 장비 장착, 장비 조합 기능이다.

첫 Unity 프로토타입의 즉시 구현 범위는 아니다. 다만 코드와 데이터 구조가 나중에 이 시스템을 붙일 수 있게 방향을 분명히 해 둔다.

## 핵심 방향

- 스테이지별 시작 첫 작업자는 플레이어 고유 작업자다.
- 이 고유 작업자는 새 스테이지에서도 플레이어 정체성과 성장감을 이어 주는 중심 캐릭터다.
- 스테이지 클리어, 목표 달성, 이벤트 보상 등으로 상자를 얻는다.
- 상자를 열면 장비 아이템이 나온다.
- 장비는 플레이어 고유 작업자에게 장착한다.
- 장비는 작업자 속도, 운반 능력, 생산 보조, 수익 보너스 같은 효율을 바꾼다.
- 장비 조합 또는 합성을 통해 같은 장비나 재료 장비를 상위 등급으로 올릴 수 있게 한다.

## 마일스톤 8 기준 설계 결정

마일스톤 8은 장기 메타 시스템을 바로 구현하는 단계가 아니라, 이후 구현이 흔들리지 않도록 1차 기준을 정하는 단계다.

현재 기준:

- 첫 장비 슬롯은 `Head`, `Body`, `Tool` 3부위로 시작한다.
- 보상 상자는 스테이지 클리어와 명확한 생산 목표 달성 보상으로 지급한다.
- 장비 장착 UI는 고유 작업자 중심의 단일 화면으로 시작한다.
- 장비 조합은 중복 장비 3개를 같은 장비의 상위 레벨로 합치는 방식부터 시작한다.
- 설계도 기반 제작은 장기 후보로 남기되, 첫 구현에는 넣지 않는다.
- 랜덤 박스는 보상감 용도이지 모바일식 과금 압박 구조로 쓰지 않는다.

첫 구현 목표는 "상자를 열고, 나온 장비를 고유 작업자에게 끼우면 다음 스테이지 시작부터 효율이 달라진다"는 경험이다.

## Eatventure 참고 범위

참고 출처:

- Eatventure Wiki `Character`: 캐릭터는 첫 요리사이며 사라지지 않고, 아이템을 장착할 수 있으며, 기본 아이템 카테고리는 Head, Body, Hand로 설명된다.  
  출처: https://eatventure.fandom.com/wiki/Character
- Eatventure Game Wiki `Clothes`: 현재 의류/장비 카테고리는 Hats, Shirts, Hand Tools, Pets, Rings, Necklaces까지 확장되어 있으며, 장비는 상자와 이벤트 보상 등으로 얻고 보너스를 제공한다.  
  출처: https://eatventure-games.fandom.com/wiki/Clothes
- Eatventure Game Wiki `Equipment Boxes`: 장비 상자는 여러 종류가 있고, 플레이 진행, 이벤트, 클럽 보상 등으로 얻을 수 있으며 장비와 설계도, 펫 관련 보상을 포함한다.  
  출처: https://eatventure-games.fandom.com/wiki/Equipment_Boxes
- Eatventure Game Wiki `Blueprints`: 높은 등급 장비는 설계도와 재료 장비를 요구하는 제작/제작식 구조를 가진다.  
  출처: https://eatventure-games.fandom.com/wiki/Blueprints

참고할 구조:

- 장비 슬롯이 나뉘어 있다.
- 보상 상자에서 장비를 얻는다.
- 낮은 등급 장비가 성장 재료가 된다.
- 일부 장비는 플레이어 자신뿐 아니라 전체 작업자 효율에도 영향을 줄 수 있다.
- 장비 수집과 조합이 스테이지 반복 플레이의 장기 목표가 된다.

그대로 가져오면 안 되는 것:

- 장비 이름, 외형, 아이콘, 수치, 등급 색상, 조합식의 직접 복제
- 모바일 과금 또는 광고 보상 구조
- 원작 캐릭터 실루엣과 의상 문법
- 특정 이벤트 장비와 이벤트 구성

현재 문서에서는 Eatventure의 기본 장비 슬롯인 Head/Body/Hand 계열을 1차 참고점으로 삼는다. 다만 Eatventure가 Pets, Rings, Necklaces 같은 추가 카테고리도 가진 상태이므로, Tiny Factory의 추가 슬롯은 첫 3부위가 재미를 만든 뒤 별도 결정한다.

## Tiny Factory 장비 부위

마일스톤 8 기준 첫 장비 슬롯은 다음 3부위로 둔다.

| 부위 | 예시 | 1차 주요 효과 | 설계 의도 |
| --- | --- | --- |
| Head | 안전모, AR 작업 고글, 검사 바이저 | 주문 인식 속도, 전체 작업자 속도 소폭 증가 | 플레이어 고유 작업자의 운영 판단/지휘 느낌을 담당 |
| Body | 작업복, 방진복, 외골격 조끼 | 고유 작업자 이동 속도, 운반 안정성 | 화면에서 체감되는 이동/운반 성장을 담당 |
| Tool | 전동 드라이버, 납땜기, 멀티툴 | 조립 속도, 픽업 가치 보너스 | 생산성과 수익 성장을 직접 담당 |

추가 후보:

- Accessory: 배지, 데이터 칩, 통신 모듈
- Drone/Pet 대응 슬롯: 보조 드론, 미니 로봇, 자동 픽업 장치

초기 구현에서는 Eatventure의 Head/Body/Hand 구조를 전자/가전 테마로 바꾼 `Head`, `Body`, `Tool` 3부위만 사용한다. 추가 슬롯은 장비 시스템이 실제 재미를 만든 뒤 확장한다.

장비 등급 초안:

| 등급 | 역할 | 1차 수치 범위 |
| --- | --- | --- |
| Common | 초반 상자에서 자주 나오는 기본 장비 | 단일 효과 `+2~5%` |
| Uncommon | 초반 체감 장비 | 단일 효과 `+5~8%` |
| Rare | 특정 성장 방향을 밀어 주는 장비 | 단일 효과 `+8~12%` 또는 보조 효과 1개 |
| Epic | 장기 목표 장비 | 강한 효과 1개와 보조 효과 1개 |

첫 구현에서는 `Common`, `Uncommon`, `Rare`까지만 데이터로 두고, `Epic`은 조합/설계도 시스템이 붙을 때 열어도 된다.

## 장비 효과 방향

장비 효과는 단순 숫자 상승만으로 끝나지 않아야 한다.

효과 후보:

- 플레이어 고유 작업자 이동 속도 증가
- 플레이어 고유 작업자 운반량 증가
- 모든 작업자 이동 속도 증가
- 특정 스테이션 조립 속도 증가
- 판매가 또는 출고 보너스 증가
- 첫 생산 또는 첫 판매까지 걸리는 시간 감소
- 스테이지 시작 자금 또는 시작 부품 보너스
- 보상 상자 추가 획득 확률 증가

첫 구현 시 우선순위:

1. 고유 작업자 이동 속도
2. 모든 작업자 속도 또는 생산 속도
3. 판매가 보너스
4. 운반량 증가

## 보상 상자

상자는 반복 플레이의 큰 보상 단위다.

획득 후보:

- 스테이지 클리어
- 특정 생산 목표 달성
- 5분 플레이 테스트 같은 단기 목표 달성
- 이벤트 납품 완료
- 첫 자동화 설비 구매 같은 마일스톤 달성

상자 등급 후보:

- Basic Box
- Advanced Box
- Premium Box
- Event Box

상자 보상 후보:

- 장비 아이템
- 장비 조합 재료
- 연구 포인트
- 소량의 시작 보너스 자원

마일스톤 8 기준 보상 상자 지급 타이밍:

| 지급 타이밍 | 상자 | 이유 |
| --- | --- | --- |
| 스테이지 클리어 | Basic Box 1개 | 핵심 반복 보상. 다음 스테이지를 시작할 이유를 만든다. |
| 첫 자동화 목표 달성 | Basic Box 1개 | 첫 직원 또는 첫 자동 설비 구매 같은 초반 목표를 강화한다. |
| 생산 목표 달성 | Basic Box 또는 Advanced Box | `보조배터리 50개 출고`처럼 제품 테마 목표를 만든다. |
| 챌린지 목표 달성 | Advanced Box | 제한 시간, 특정 업그레이드 제한 같은 변주용 보상이다. |
| 이벤트 납품 완료 | Event Box | 장기 이벤트용. 첫 구현에서는 보류한다. |

첫 구현 권장:

```text
첫 스테이지 클리어 조건: 픽업/출고 완료 30회 + 직원 2명 보유
보상: Basic Box 1개
추가 목표: 픽업/출고 완료 50회
보상: Basic Box 1개
```

이 조건은 현재 프로토타입의 5분 재테스트 결과와 맞물린다. 마일스톤 7A에서 5분 후 픽업/출고 완료가 `25회`였으므로, `30회`는 첫 클리어 목표로 약간 더 밀어붙이는 수준이다.

상자 구성 초안:

| 상자 | 내용물 | 용도 |
| --- | --- | --- |
| Basic Box | Common 장비 위주, 낮은 확률 Uncommon | 첫 장비 획득과 중복 누적 |
| Advanced Box | Uncommon 위주, 낮은 확률 Rare | 목표 달성 보상 |
| Premium Box | Rare 이상 후보 | 첫 구현 보류 |
| Event Box | 이벤트 전용 장비 또는 재료 | 첫 구현 보류 |

확률은 아직 최종 수치로 두지 않는다. 첫 구현에서는 상자별 후보 풀을 작게 두고, 체감 테스트 후 확률을 정한다.

첫 프로토타입 본편 루프에는 상자를 아직 넣지 않는다. 다음 단계에서 메타 구현을 시작한다면 스테이지 클리어 판정, 상자 지급, 장비 인벤토리의 최소 데이터 구조부터 넣는다.

## 장비 장착 UI 초안

장비 UI는 고유 작업자 중심으로 시작한다. 메뉴 이름은 임시로 `Worker Gear`를 사용한다.

기본 화면 구성:

```text
상단: Player Worker_01 이름과 현재 총 보너스 요약
좌측: 고유 작업자 미리보기
중앙: Head / Body / Tool 장착 슬롯
우측: 보유 장비 목록
하단: 선택 장비 상세, Equip 버튼, Combine 버튼
```

장비 목록 필터:

- 전체
- Head
- Body
- Tool
- 조합 가능

장비 상세에 표시할 정보:

- 장비 이름
- 부위
- 등급
- 레벨
- 현재 효과
- 다음 레벨 효과
- 조합에 필요한 중복 수량

장착 흐름:

1. 플레이어가 `Worker Gear` 화면을 연다.
2. 장비 슬롯 또는 보유 장비를 선택한다.
3. 장비 상세를 확인한다.
4. `Equip` 버튼을 누르면 해당 부위 기존 장비와 교체된다.
5. 총 보너스 요약이 갱신된다.
6. 다음 스테이지 또는 현재 테스트 씬에 보너스가 적용된다.

첫 UI 구현에서는 드래그 앤 드롭을 넣지 않는다. 클릭 선택과 `Equip` 버튼만 사용한다.

첫 구현에서 표시할 보너스 요약:

```text
Player Worker Speed +8%
All Worker Speed +3%
Assembly Speed +5%
Pickup Value +5%
```

실제 효과 연결은 `SupportBonusSlots`와 `WorkerManager`, `AssemblyBench`, `PickupCounter` 쪽에 단계적으로 붙인다.

## 장비 조합

조합 기능의 목표는 중복 장비를 의미 있는 성장 재료로 바꾸는 것이다.

기본 방향:

- 같은 장비 여러 개를 합쳐 레벨을 올린다.
- 특정 등급 이상은 하위 장비 또는 조합 재료를 요구한다.
- 조합 결과는 예측 가능해야 하며, 과도한 랜덤성에 의존하지 않는다.
- Steam PC 게임에서는 모바일식 과금 압박보다 목표 달성형 조합을 우선한다.

조합식 예시:

```text
같은 Common 장비 3개 -> 강화된 Common 장비
같은 부위 Rare 장비 3개 -> Epic 조합 재료
특정 Blueprint + 재료 장비 -> 고급 장비 제작
```

마일스톤 8 기준 1차 조합 규칙:

```text
같은 장비 3개 -> 같은 장비 레벨 +1
장비 최대 레벨은 등급별로 제한
최대 레벨 장비의 중복은 향후 승급 재료로 보관
```

등급별 레벨 상한 초안:

| 등급 | 최대 레벨 | 조합 규칙 |
| --- | --- | --- |
| Common | 5 | 같은 장비 3개로 +1 |
| Uncommon | 10 | 같은 장비 3개로 +1 |
| Rare | 15 | 같은 장비 3개로 +1 |
| Epic | 20 | 첫 구현 보류 |

예시:

```text
Safety Goggles Lv 1 3개 -> Safety Goggles Lv 2 1개
Power Driver Lv 4 3개 -> Power Driver Lv 5 1개
Common 최대 레벨 장비 중복 -> 승급 재료 후보로 보관
```

설계도 기반 제작은 다음 단계 이후 후보로 둔다.

```text
Blueprint + 같은 부위 Rare 장비 + 조합 재료 -> Epic 장비
```

첫 구현에서 설계도를 제외하는 이유:

- 상자, 인벤토리, 장착, 단순 중복 조합만으로도 충분히 검증할 내용이 많다.
- 설계도는 UI 설명량과 데이터 구조를 크게 늘린다.
- Steam PC용으로는 예측 가능한 목표형 조합이 먼저 필요하다.

## 첫 메타 구현 데이터 구조 초안

다음 구현 단계에서 필요한 최소 데이터 후보:

```text
EquipmentDefinition
- id
- displayName
- slot
- rarity
- baseEffects
- maxLevel

EquipmentInstance
- definitionId
- level
- quantity or instanceId

RewardBoxDefinition
- id
- displayName
- rewardPool

RewardEntry
- equipmentDefinitionId
- weight
- minLevel
- maxLevel

PlayerMetaState
- ownedEquipment
- equippedHead
- equippedBody
- equippedTool
- unopenedBoxes
```

첫 저장 구현 전까지는 메모리 상태로만 테스트해도 된다. 저장을 시작할 때는 이 구조를 JSON 저장 후보로 검토한다.

## 첫 프로토타입과의 관계

첫 프로토타입에는 다음을 넣지 않는다.

- 보상 상자
- 장비 장착 UI
- 장비 효과
- 장비 조합
- 장비 인벤토리

첫 프로토타입에서 준비할 수 있는 것:

- 시작 작업자를 고유 작업자로 해석할 수 있는 명명과 구조
- `WorkerController`가 장비 보너스를 나중에 받을 수 있게 속도/운반량을 필드로 분리
- 돈/업그레이드 시스템이 장비 보너스와 충돌하지 않게 설계

## 미정 사항

- 추가 장비 슬롯을 열지 여부
- Eatventure 참고 슬롯을 어느 정도까지 확장 반영할지
- Epic 이상 장비 등급 체계
- 설계도 기반 고급 조합식
- 상자 획득 빈도
- 스테이지별 실제 클리어 조건
- 장비 효과가 플레이어 고유 작업자 전용인지, 전체 작업자에도 적용되는지
