# Eatventure 레퍼런스 분석 초안

## 문서 목적

이 문서는 Eatventure의 기본 시스템 이해를 바로잡기 위한 기준 문서다. 이전 문서 일부에는 플레이어가 직접 캐릭터를 WASD로 움직이고 E 키로 상호작용하는 전제가 섞여 있었는데, 이는 Eatventure식 운영 게임의 핵심과 맞지 않는다.

이 프로젝트에서 참고할 Eatventure의 핵심은 직접 이동 조작이 아니라, 스테이션을 열고 업그레이드하며 직원과 보조 시스템이 자동으로 돈을 벌게 만드는 운영 흐름이다.

## Eatventure의 핵심 시스템 이해

### 1. 플레이어 역할

플레이어는 캐릭터를 직접 조종하는 액션 주체가 아니라 운영자에 가깝다. 플레이어의 핵심 행동은 다음과 같다.

- 스테이션을 해금한다.
- 스테이션 레벨을 올린다.
- 업그레이드를 산다.
- 직원, 계산원, 보조 시스템을 해금한다.
- 임시 보상, 헬퍼, 이벤트, 장비, 영구 보너스를 선택한다.
- 어떤 곳에 먼저 투자할지 결정한다.

우리 게임에서 플레이어는 작업장 안을 WASD로 돌아다니는 캐릭터가 아니라, 작은 공장의 사장 또는 운영 관리자다.

### 2. 코어 루프

Eatventure의 기본 루프는 다음 구조로 이해한다.

1. 고객 또는 주문 수요가 발생한다.
2. 스테이션이 상품을 생산한다.
3. 직원 또는 자동 행동자가 상품을 고객에게 전달한다.
4. 고객이 현금을 지불한다.
5. 플레이어가 현금으로 스테이션, 직원, 수익 배율, 속도, 자동화 요소를 업그레이드한다.
6. 더 많은 스테이션과 더 큰 장소가 열린다.

중요한 점은 조작 루프가 "내가 직접 들고 간다"가 아니라 "자동 시스템이 돈을 벌고, 나는 병목에 투자한다"는 것이다.

### 3. 성장 축

Eatventure의 성장 축은 여러 겹으로 겹친다.

- 스테이션 해금: 새 음식 또는 새 생산 지점이 열린다.
- 스테이션/제품 레벨업: 같은 음식 또는 상품의 수익성이 오른다.
- 직원/캐셔 고용: 처리와 전달이 자동화된다.
- 업그레이드 구매: 특정 음식, 전체 수익, 작업 속도, 주문 속도 등이 오른다.
- 장소 진행: 작은 매장에서 큰 매장으로 넘어가며 스테이션 수와 최대 레벨이 늘어난다.
- 메타 보너스: Vault, 장비, 박스, 장비 조합, 이벤트, 오프라인 수익이 장기 성장을 만든다.

우리 게임은 이를 전자/가전 생산에 맞춰 다음처럼 바꾼다.

- 생산 스테이션 해금: 부품대, 조립대, 검사대, 포장대, 출고대
- 제품 레벨업: 보조배터리 같은 제품의 출고 단가 상승
- 스테이션 레벨업: 처리 속도, 보관량, 출고량 상승
- 작업자 자동화: 조립공, 물류 담당, 검사원, 포장 담당, 출고 담당
- 설비 자동화: 컨베이어, 로봇 팔, 자동 검사기, 자동 포장기
- 장소 진행: 작업 코너, 수리점, 소형 조립소, 자동화 라인, 스마트 팩토리
- 메타 보너스: 브랜드 신뢰도, 기술 연구, 공급 계약, 작업자 장비 모듈

### 4. 주변 시스템

Eatventure의 재미는 코어 루프만이 아니라 주변 시스템에서 반복적으로 개입할 이유를 만든다는 점에 있다.

- 팁: 고객 판매 외의 작은 보너스 수익.
- 투자자/배달 기사: 일시적 큰 보상 또는 특정 납품 요청.
- 헬퍼: 일정 시간 동안 작업을 도와주는 임시 고성능 작업자.
- 영구 보너스 카드: 주문 속도, 고객 이동 속도, 팁 확률, 오프라인 수익, 투자자 보상 등 반복 플레이의 효율을 올림.
- 장비와 박스: 보상 상자를 열어 장비를 얻고, 장비 장착과 성장으로 반복 플레이의 장기 목표를 만듦.
- 장비 조합: 중복 장비나 재료 장비를 상위 성장으로 바꾸는 장기 진행 장치.
- 오프라인 수익: 게임을 꺼도 일정 수준의 진행감 제공.
- 이벤트: 기본 루프를 다른 테마와 보상 구조로 변주.

우리 게임의 대응 후보는 다음과 같다.

| Eatventure 요소 | Tiny Factory 치환 후보 | 프로토타입 반영 여부 |
| --- | --- | --- |
| 팁 | 고객 감사 보너스, 빠른 출고 보너스, 리뷰 보너스 | 1차에서는 보류, UI 슬롯만 고려 |
| 투자자 | 대량 구매자, 부품 거래상, 긴급 계약 제안 | 1차에서는 보류 |
| 배달 기사 | 납품 트럭, 픽업 기사, 기업 출고 요청 | 1차 이후 후보 |
| 헬퍼 | 임시 숙련 기술자, 파견 엔지니어, 자동화 컨설턴트 | 1차 이후 후보 |
| Vault | 연구실, 운영 매뉴얼, 영구 공정 개선 | 1차 이후 후보 |
| 장비 | 고유 작업자 장비, 공구, 작업복, 보조 드론 | 장기 메타 후보 |
| 박스 | 스테이지 클리어 보상 상자, 계약 완료 상자 | 장기 메타 후보 |
| 장비 조합 | 중복 장비 합성, 설계도 기반 제작 | 장기 메타 후보 |
| 오프라인 수익 | 야간 생산, 무인 라인 수익 | 저장 도입 이후 후보 |
| 이벤트 | 신제품 출시, 부품 부족, 박람회 납품, 리콜 대응 | 1차 이후 후보 |

## 프로토타입 방향 수정

### 이전 전제

- 플레이어 캐릭터가 이동한다.
- 플레이어가 직접 부품을 집고 조립대에 넣는다.
- WASD와 E 키로 루프를 검증한다.
- 직원은 플레이어의 반복 작업 일부를 나중에 대체한다.

### 수정 전제

- 플레이어는 작업자를 직접 조종하지 않는다.
- 작업자는 시작부터 자동으로 이동하고 생산한다.
- 플레이어는 마우스 클릭 또는 UI 버튼으로 스테이션 해금, 레벨업, 우선순위, 자동화 구매를 한다.
- 프로토타입의 첫 검증 질문은 조작감이 아니라 "자동 생산 흐름이 읽히고 투자 판단이 생기는가"다.
- 스테이지별 시작 첫 작업자는 플레이어 고유 작업자로 본다.
- 장기적으로는 스테이지 클리어 보상 상자에서 장비를 얻고, 고유 작업자에게 장착하며, 장비 조합으로 성장시키는 구조를 둔다.

## 첫 프로토타입에 즉시 반영할 기준

1. Milestone 2는 `플레이어 이동`이 아니라 `자동 작업 루프 골격`이어야 한다.
2. 첫 씬에는 `Player`보다 `Operator` 또는 `GameManager` 관점의 클릭/UI 입력이 우선이다.
3. 작업자는 처음부터 존재하거나 첫 업그레이드로 바로 고용 가능해야 한다.
4. 생산 루프는 자동으로 돌아가야 하며, 플레이어는 업그레이드로 속도와 수익을 바꿔야 한다.
5. WASD, E 키, 근접 상호작용, 들고 있는 아이템 UI는 1차 프로토타입 범위에서 제거한다.
6. `팁`, `헬퍼`, `대량 구매자/투자자`, `장비`, `보상 상자`, `조합`은 1차 구현 범위에는 넣지 않되, 설계 슬롯으로 남긴다.

## 구현 우선순위 수정안

1. 스테이션 배치: 부품대, 조립대, 판매/출고대
2. 작업자 1명 생성
3. 작업자 상태 머신: 부품 픽업, 조립대 투입, 완성품 회수, 출고
4. 조립대 타이머와 제품 생성
5. 자동 판매와 현금 증가
6. 스테이션 레벨업 UI
7. 작업자 속도 또는 조립 속도 업그레이드
8. 새 작업자 또는 자동화 설비 구매
9. 팁/헬퍼/계약 같은 주변 시스템 후보 검토
10. 장비/보상 상자/조합 메타는 `docs/12-meta-progression-design.md`에서 별도 설계

## 주의할 점

- Eatventure의 UI, 수치, 이름, 광고 보상 구조를 그대로 옮기지 않는다.
- 단순히 음식 이름을 전자제품 이름으로 바꾸는 방식은 피한다.
- 하지만 "스테이션이 돈을 벌고, 플레이어는 업그레이드와 자동화에 투자한다"는 작동 원리는 핵심 레퍼런스로 삼는다.
- PC용 Steam 게임으로 확장할 때는 마우스 중심 조작, 더 넓은 정보 패널, 생산 라인 병목 표시, 자동화 설비의 시각적 재미를 강화한다.
