# 첫 프로토타입 범위 정의서

## 문서 목적

이 문서는 Unity로 만들 첫 프로토타입의 범위, 제외 항목, 검증 질문, 성공 기준을 정의한다. 목표는 최종 게임의 축소판을 만드는 것이 아니라, 이 게임의 핵심 재미가 성립하는지 가장 작은 단위로 확인하는 것이다.

현재 프로토타입은 조사 완료 후 확정된 설계가 아니라, 기존 초기 기획 문서를 바탕으로 한 가설 검증용이다.

## 프로토타입 한 줄 목표

작업자와 스테이션이 자동으로 부품을 옮기고 제품을 조립, 판매하며, 플레이어는 스테이션 해금과 업그레이드로 생산 흐름을 키우는 핵심 운영 루프를 Unity에서 직접 체감한다.

## 검증하려는 핵심 가설

1. 전자/가전 생산 테마는 음식 조리 테마처럼 직관적으로 이해될 수 있다.
2. 자동 부품 공급, 조립, 판매, 업그레이드의 짧은 루프는 반복 플레이의 기본 재미를 만들 수 있다.
3. 작업자와 설비의 자동 처리량이 늘어나는 순간은 충분한 성장 체감을 만든다.
4. 위에서 수직으로 내려다보는 단순한 3D 탑다운 표현만으로도 생산 라인의 변화와 자동화 쾌감을 전달할 수 있다.
5. Steam PC용 게임으로 확장하려면 어떤 조작, UI, 자동화 깊이가 필요한지 초기 판단을 할 수 있다.

## 프로토타입 기준 플레이 경험

플레이어는 작은 전자제품 작업장을 수직 탑다운 시점으로 내려다보며 운영한다. 작업장에는 부품 보관대, 조립대, 픽업/출고대, 제품 레벨업 버튼, 직원 구매 버튼, 설비 구매 버튼이 있다.

스테이지별 첫 작업자는 플레이어 고유 작업자다. 이 작업자는 부품 보관대에서 부품을 들고 조립대로 이동한다. 조립대에 부품이 들어가면 일정 시간 후 간단한 제품이 완성된다. 완성품이 픽업/출고대로 이동하면 돈을 얻는다. 플레이어는 돈을 모아 제품 레벨, 조립 속도, 작업자 속도, 스테이션 레벨을 업그레이드한다. 제품 레벨이 오르면 같은 출고 1회당 수익이 증가하고, 이 단가 상승으로 추가 직원 또는 자동화 설비 구매 시간이 압축되어야 한다.

이 흐름이 5~10분 안에 반복되고, 플레이어는 업그레이드 전후의 차이를 체감해야 한다.

## 포함 범위

### 1. 기본 씬

포함:

- 하나의 테스트 작업장 씬
- 바닥, 벽 또는 경계
- 수직 탑다운 고정 카메라
- 조명
- 자동 작업자 캐릭터
- 부품 보관대
- 조립대
- 판매대
- 업그레이드 지점 또는 UI
- 직원 1명

제외:

- 여러 공장
- 여러 층 또는 방
- 완성된 인테리어
- 최종 아트 리소스

### 2. 운영 입력

포함:

- 마우스 클릭 또는 UI 버튼 기반 운영 입력
- 수직 탑다운 고정 카메라
- 스테이션 선택, 레벨업, 직원 구매, 업그레이드 구매

기본 가정:

- 초기에는 WASD 이동을 사용하지 않는다.
- 플레이어는 직접 작업자를 조종하지 않는다.
- 조작감 검증보다 자동 생산 흐름의 가독성과 업그레이드 판단을 우선한다.

### 3. 아이템 운반

포함:

- 작업자가 부품 1개를 들 수 있다.
- 작업자가 완성품 1개를 들 수 있다.
- 작업자가 들고 있는 물건이 시각적으로 표시된다.
- 들고 있는 상태에 따라 작업자 다음 목적지가 달라진다.

제외:

- 플레이어 직접 운반
- 플레이어 인벤토리
- 아이템 스택
- 복잡한 물류 우선순위

### 4. 생산 루프

포함:

- 부품 보관대에서 부품 생성 또는 무한 제공
- 조립대에 부품 투입
- 조립 시간 대기
- 완성품 생성
- 완성품을 판매대로 이동
- 판매 시 돈 획득

초기 제품:

- 가칭: `Basic Gadget`
- 의미: 스마트폰, 이어폰, 보조배터리 중 하나로 확정하기 전 사용하는 임시 전자제품
- 프로토타입에서는 외형보다 루프 검증을 우선한다.

제외:

- 제품군 다중화
- 부품 종류 다중화
- 검사 공정
- 포장 공정
- 결함률
- 품질 등급

### 5. 경제와 업그레이드

포함:

- 돈 표시
- 제품 판매가
- 제품 레벨업
- 조립대 업그레이드 1종
- 판매가 업그레이드 1종 또는 조립 속도 업그레이드 1종
- 직원 구매 비용
- 업그레이드 구매 후 즉시 효과 반영

초기 수치 가설:

| 항목 | 초기값 |
| --- | --- |
| 제품 판매가 | 5 |
| 제품 레벨업 비용 | 10 |
| 제품 레벨업 효과 | 출고 가치 증가 |
| 조립 시간 | 3초 |
| 조립 속도 업그레이드 비용 | 20 |
| 조립 속도 업그레이드 효과 | 조립 시간 20% 감소 |
| 판매가 업그레이드 비용 | 30 |
| 판매가 업그레이드 효과 | 판매가 +5 |
| 직원 구매 비용 | 50 |

수치는 재미 검증용 임시값이다. 최종 밸런스로 보지 않는다.

### 6. 작업자와 자동화

포함:

- 작업자 1명 이상이 자동 반복 작업을 수행한다.
- 스테이지 시작 작업자 1명은 플레이어 고유 작업자로 간주한다.
- 추가 직원 또는 자동화 설비를 구매할 수 있다.
- 첫 버전 작업자 역할은 하나만 선택한다.

권장 직원 역할:

- 운반/조립 작업자: 부품 보관대에서 부품을 가져와 조립대에 넣고, 완성품을 판매대 또는 출고대로 옮긴다.

선택 이유:

- 화면에서 자동 생산 흐름을 가장 빠르게 보여준다.
- 조립대가 계속 돌아가는 느낌을 만들기 쉽다.
- 판매까지 자동화하되, 플레이어는 업그레이드와 병목 해소로 개입한다.

제외:

- 복수 역할 직원
- 직원 레벨업
- 직원 역할 변경 UI
- 직원 경로 최적화
- 매니저 시스템
- 고유 작업자 장비 효과

### 7. UI/HUD

포함:

- 현재 돈
- 현재 선택한 스테이션
- 조립대 상태
- 업그레이드 비용
- 직원 구매 비용
- 간단한 안내 텍스트

제외:

- 최종 HUD 디자인
- 복잡한 메뉴
- 옵션 화면
- 세이브 슬롯 UI
- 튜토리얼 시스템

### 8. 저장

첫 프로토타입에서는 저장 기능을 제외한다.

이유:

- 핵심 재미 검증에는 저장이 필요하지 않다.
- 저장 구조를 성급히 만들면 이후 데이터 구조 변경 비용이 생긴다.
- 프로토타입 1차 목표는 5~10분 플레이 루프 확인이다.

단, 두 번째 프로토타입부터는 로컬 JSON 저장을 검토한다.

- 장비 장착 UI
- 장비 조합
- 이벤트
- 메타 진행
- 사운드 완성
- 최종 아트

## 프로토타입 성공 기준

### 최소 성공

다음이 가능하면 최소 성공으로 본다.

- Unity에서 씬이 실행된다.
- 작업자가 자동으로 이동할 수 있다.
- 작업자가 부품을 가져와 조립대에 넣을 수 있다.
- 조립 시간이 지난 뒤 제품이 나온다.
- 제품이 판매대 또는 출고대로 이동하면 돈이 증가한다.
- 돈으로 업그레이드를 구매할 수 있다.
- 작업자 1명이 자동으로 반복 작업을 수행한다.

### 의미 있는 성공

다음이 느껴지면 의미 있는 성공으로 본다.

- 플레이어가 1분 안에 무엇을 업그레이드해야 하는지 이해한다.
- 업그레이드 후 작업 속도 또는 수익 차이가 체감된다.
- 직원 구매 후 처리량 증가와 자동화 확장이 느껴진다.
- 화면에서 생산 흐름이 읽힌다.
- 수직 탑다운 시점에서도 각 스테이션과 작업자 상태가 구분된다.
- 5분 이상 반복해도 다음 업그레이드를 사고 싶다는 동기가 있다.

### 실패로 볼 신호

다음이 나타나면 설계 재검토가 필요하다.

- 부품, 조립, 판매 관계가 직관적으로 이해되지 않는다.
- 이동이 지루하고 조작 피로가 먼저 느껴진다.
- 초반 자동 작업 흐름이 너무 단조롭다.
- 직원 구매 후에도 플레이어의 투자 판단이 생기지 않는다.
- 업그레이드 효과가 숫자로만 느껴지고 행동 변화가 없다.
- 전자/가전 테마가 음식 게임의 스킨처럼 보인다.

## 테스트 플레이 체크리스트

테스트 후 다음 질문에 답한다.

- 첫 30초 안에 목표가 보였는가?
- 첫 판매까지 시간이 너무 길거나 짧지 않았는가?
- 조립 시간은 기다릴 만했는가?
- 업그레이드를 사고 싶다는 생각이 들었는가?
- 직원 구매 전후 처리량 차이가 명확했는가?
- 자동화 관찰과 운영 개입의 비율이 적절했는가?
- 화면에서 어떤 오브젝트가 어떤 역할인지 알 수 있었는가?
- 첫 작업자가 플레이어 고유 작업자로 느껴졌는가?
- 전자제품을 만든다는 느낌이 있었는가?
- 10분 뒤에도 확장하고 싶은 마음이 있었는가?
- 다음에 추가해야 할 시스템이 자연스럽게 떠올랐는가?

## 첫 구현 순서 제안

1. Unity 프로젝트 생성
2. 기본 씬 생성
3. 부품 보관대, 조립대, 판매대 배치
4. 작업자 1명 자동 이동 구현
5. 작업자 아이템 운반 구현
6. 조립대 타이머와 제품 생성 구현
7. 판매대와 돈 획득 구현
8. HUD 구현
9. 스테이션 레벨업 또는 업그레이드 1종 구현
10. 추가 직원 또는 자동화 설비 구매 구현
11. 테스트 플레이 후 수치 조정

- 폴더 구조 제안

- 오류 메시지 분석
- 다음 수정안 작성

## 미정 사항

다음은 구현 전 또는 구현 중 사용자 확인이 필요할 수 있다.

- Unity 정확한 버전
- 프로젝트 이름
- 초기 제품을 스마트폰, 이어폰, 보조배터리 중 무엇으로 표현할지
- UI를 uGUI로 시작할지 UI Toolkit으로 시작할지
- 플레이어 입력을 스테이션 클릭 중심으로 할지, 하단 UI 버튼 중심으로 할지
- 장비/상자/조합을 몇 번째 프로토타입부터 구현할지
