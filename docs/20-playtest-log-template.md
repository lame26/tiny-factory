# 플레이테스트 기록 템플릿

## 문서 목적

이 문서는 첫 Unity 프로토타입을 실행한 뒤 플레이 감각을 기록하기 위한 템플릿이다. 목표는 "재밌다/재미없다" 같은 인상만 남기는 것이 아니라, 어떤 루프가 작동했고 어떤 부분이 막혔는지 다음 수정 작업으로 이어지게 만드는 것이다.

## 사용 시점

다음 상황에서 사용한다.

- 첫 루프가 Unity에서 실행된 뒤
- 새 기능을 추가한 뒤
- 수치를 조정한 뒤
- 작업자 자동 루프가 들어간 뒤
- 운영 입력 방식이나 카메라를 바꾼 뒤
- 수직 탑다운 시점으로 바꾼 뒤
- 고유 작업자, 장비, 보상 상자 같은 메타 시스템을 붙인 뒤

기록은 길게 쓰지 않아도 된다. 중요한 것은 같은 기준으로 반복 기록하는 것이다.

점수 기준:

- 1: 거의 작동하지 않음

- 3: 기본은 되지만 밋밋함
- 4: 의도한 재미가 어느 정도 느껴짐
- 5: 명확하게 재미 또는 가능성이 느껴짐
