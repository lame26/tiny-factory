# 첫 프로토타입 작업 보드

## 문서 목적

이 문서는 Unity 프로젝트 생성 후 바로 진행할 작업을 순서대로 관리하기 위한 작업 보드다. 작업이 끝날 때마다 상태를 갱신하고, 중요한 결정은 `docs/work-history.md`에도 요약한다.

상태 표기:

- `TODO`: 아직 시작하지 않음
- `DOING`: 진행 중
- `DONE`: 완료
- `BLOCKED`: 막힘

## 우선순위 규칙

작업 중 판단 기준:

- 먼저 자동 루프가 돌아가야 한다.
- 예쁜 화면보다 역할 구분이 중요하다.
- 첫 자동화는 작업자 1명으로 충분하다.
- 첫 작업자는 플레이어 고유 작업자로 해석될 수 있게 둔다.
- 새 제품군은 첫 루프가 재미있을 때 추가한다.
- 저장, Steam, 메타 시스템은 1차 테스트 이후로 미룬다.
