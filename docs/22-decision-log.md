## 현재 보류한 사항

| 항목 | 보류 이유 |
| --- | --- |
| 최종 출시명 | Steam 검색성, 상표, 유사 게임 검토 필요 |
| Steamworks 연동 | 첫 프로토타입 검증에는 불필요 |
| 저장 시스템 | 데이터 구조가 안정된 뒤 설계하는 것이 안전 |
| 여러 제품군 구현 | 첫 루프 검증 후 확장 |
| 연구/메타 시스템 | 핵심 루프와 자동화 체감 검증 후 설계 |
| 최종 아트 스타일 | 조작/루프 검증이 우선 |
| BM/가격/얼리 액세스 | 시장 조사와 프로토타입 반응 이후 판단 |

## 이름 관련 판단

`Tiny Factory`는 내부 가칭으로 채택했다.

장점:

- 짧고 기억하기 쉽다.
- `Factory`가 장르 기대치를 빠르게 전달한다.
- 초반 작은 업장과 후반 자동화 공장 판타지를 모두 담을 수 있다.
- "내 컴퓨터 속 작은 공장"이라는 해석이 좋다.

주의:

- 일반적인 단어 조합이라 검색성과 고유성이 약할 수 있다.
- 최종 출시명으로 쓰기 전 Steam, Google, 상표, 유사 앱 검토가 필요하다.
- 최종명은 `Tiny Factory: Gadget Works`처럼 부제를 붙이는 방향도 열어둔다.

## 업장 progression 현재 기준

```text
전자제품 악세사리 판매점
→ 동네 수리/조립점
→ 중고/리퍼비시 샵
→ 브랜드 전문 AS점
→ 온라인 주문 처리 센터
→ 소형 전자제품 조립 공장
→ 자동화 생산 라인
→ 스마트 가전 생산 공장
→ 글로벌 납품 허브
→ 연구 개발 및 브랜드 캠퍼스
```

## 다음 의사결정 후보

1. 첫 제품을 보조배터리, 스마트폰, 임시 `Basic Gadget` 중 무엇으로 할지 결정
2. Unity 설치 후 실제 버전 기록
3. Unity 프로젝트를 `unity/TinyFactoryPrototype/`에 만들지 최종 확인
4. 첫 프로토타입 카메라를 탑다운에 가깝게 할지 아이소메트릭에 가깝게 할지 결정
5. 첫 UI를 uGUI로 갈지 UI Toolkit으로 갈지 결정

## 되물어야 하는 결정

AI 작업자는 다음을 사용자 확인 없이 확정하지 않는다.

- 엔진 변경
- 목표 플랫폼 변경
- 핵심 루프 변경
- 게임명 최종 확정
- 상품 전략 결정
- 프로토타입 범위 대폭 확대
- 저장/온라인/Steam 연동 구조 도입
- 기존 핵심 문서 방향을 뒤집는 수정
